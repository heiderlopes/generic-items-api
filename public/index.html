CHECKPOINT FLUTTER

Instruções Gerais
O projeto deve estar funcionando na hora da correção.
Os métodos e widgets solicitados serão avaliados como certo ou errado (não haverá pontuação parcial dentro de cada item).
Não será permitida a consulta a ferramentas de Inteligência Artificial (IA) durante a prova. Caso haja algum indício do uso a nota será ZERO.
Caso o projeto não compile ou apresente erros que impeçam a execução, a avaliação será considerada ZERO.

Crie o projeto rodando o seguinte comando (substituindo XXXX pelo seu RM). Caso não seja gerado o package corretamente, a nota da avaliação será ZERO:
flutter create --org br.com.seurm checkpoint05_webservices_rmXXXX

Obrigatório o envio do arquivo zipado contendo:
- main.dart
- pubspec.yaml

Requisitos:
(3,0 pontos) LISTA DE REGISTROS
- Exibir os registros cadastrados na API pelo RM em uma lista.
- Cada item da lista deve mostrar os campos obrigatórios cadastrados no formulário.

(1,0 pontos) BOTÃO PARA REGISTRO
- Adicionar um Button ou FloatActionButton para adicionar um novo livro.

(3,0 pontos) EXCLUSÃO DE REGISTRO
- Adicione um botão no item da lista para realizar a exclusão do item. Após a exclusão recarregar a lista.

(3,0 pontos) FORMULÁRIO DE REGISTRO
- Você deve criar um formulário obrigatoriamente com pelo menos 4 campos: Imagem (URL), título, descrição e valor.
- Exibir preview da imagem abaixo do campo de URL, renderizando a imagem da internet.
- Botão para Salvar.

API Backend:
https://generic-items-api-a785ff596d21.herokuapp.com

GET /api/{SEU_TEMA}/{rm}
POST /api/{SEU_TEMA}
DELETE /api/{SEU_TEMA}/{rm}
